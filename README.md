# **ASICam \- ZWO ASI Camera Stream & Record 🌠**

ZWO ASIカメラを使用した天体・流星観測に特化した、RTSPライブ配信および自動分割録画アプリケーションです。  
スティックPCのような省電力環境での長期間の無人稼働を前提に設計されており、ハードウェアエンコードの自動判別やノンブロッキングでのファイル自動削除など、システムの安定性と低負荷を追求しています。

## **✨ 主な機能 (Features)**

* **ZWO ASIカメラのフルコントロール**  
  * ゲイン、露出、ホワイトバランス、ガンマなどの手動/自動調整。  
  * センサー温度や冷却パワーの監視（冷却カメラ対応）。  
  * 帯域制御、HighSpeedモードの切り替え。  
* **RTSPライブ配信 ＆ ローカル録画**  
  * FFmpegを活用した高画質・低遅延な映像パイプライン。  
  * **\[録画\]** 時計と連動した自動ファイル分割（1, 5, 10, 20, 30, 40, 50, 60分から選択可能）。  
* **インテリジェントなエンコーダ自動判定**  
  * PCに搭載されているGPU（Intel QSV / NVIDIA NVENC / AMD AMF）を自動検知し、最適なハードウェアエンコーダを適用してCPU負荷と発熱を劇的に抑えます。  
  * もちろん設定画面から手動でエンコーダを強制指定することも可能です。  
* **無人稼働のためのフェイルセーフ設計**  
  * **自動お掃除機能:** 設定した保存日数（1〜7日）を過ぎた古い録画ファイルを1時間ごとに自動で完全削除。メインの映像パイプラインに一切影響を与えない別スレッド（ノンブロッキング）処理を採用。  
  * **容量監視:** ドライブの空き容量が10GBを下回ると録画を安全に緊急停止し、クラッシュを防ぎます。  
* **タイムスタンプ ＆ 画像補正**  
  * カメラの解像度に合わせて自動リサイズされる見やすいタイムスタンプ刻印。

## **💻 動作環境 (Requirements)**

* Windows 10 / 11 (64bit)  
* ZWO ASI カメラ  
* スティックPCやミニPCでも快適に動作します（Intel QSV等のハードウェアエンコーダ推奨）

## **🔧 使用ライブラリ・ツール (Dependencies)**

本アプリをビルド・実行するには、実行ファイルと同じフォルダに以下のツールが配置されている必要があります。

* [**OpenCvSharp**](https://github.com/shimat/opencvsharp)**:** 映像の取得と画像処理  
* [**FFmpeg**](https://ffmpeg.org/)**:** 映像のエンコード（ffmpeg.exe が必要）  
* [**MediaMTX**](https://github.com/bluenviron/mediamtx)**:** RTSPサーバー機能（mediamtx.exe が必要）  
* **ASI Camera SDK:** ZWO公式のカメラ制御用DLL

## **🚀 使い方 (How to Use)**

### **1\. カメラの接続**

アプリを起動すると、PCに接続されているASIカメラが自動的にリストアップされます。カメラを選択して「接続」ボタンをクリックしてください。

### **2\. カメラパラメータの調整**

画面上のスライダーを使って、ゲインや露出時間を調整します。各項目は「Auto」チェックボックスで自動調整に任せることも可能です。必要に応じてプレビュー画面を表示してピントや画角を確認してください。

### **3\. 配信 / 録画設定**

* **保存先と日数の設定:** 「保存先設定」ボタンから、録画ファイルの保存先フォルダと、ファイルを保持する日数、分割単位を設定します。  
* 画面下の「RTSP配信」または「ローカル録画」のチェックボックスをONにすると、指定した処理がスタートします（両方を同時にONにすることはできません）。

### **4\. 詳細設定 (エンコーダの指定)**

「詳細設定」ボタンから、使用する動画エンコーダを確認・変更できます。  
通常は「自動判定」で問題ありませんが、環境に合わせて任意のハードウェアエンコーダを強制指定することも可能です（未搭載のGPUを選択した場合は、安全のため自動的にソフトウェアエンコードへフォールバックします）。

## **⚠️ 注意事項 (Notes)**

* 録画の自動削除機能は、システムの安定化のためにごみ箱を経由せず**完全削除**されます。必要な録画データは、自動削除される前に別ドライブへ退避してください。  
* スティックPC等で運用する場合、ハードウェアエンコーダが有効になっていても本体が熱くなる場合があります。必要に応じて小型USBファン等での物理的な冷却をおすすめします。
